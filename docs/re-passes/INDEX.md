# MOSVIEW total-decomp pass — coverage index

State of the multi-session reverse-engineering pass driven by the plan at
`.claude/plans/serene-knitting-corbato.md`.

Two layers of coverage:

1. **Bulk plate** — every function in scope has a non-empty PLATE comment
   carrying its signature, caller/callee counts, dominant Win32 / project
   imports, string-ref counts, and a sample. Generated by
   `scripts/auto-plate.py`. Hand-written plates are preserved.

2. **Deep annotation** — function renamed-or-justified, params/locals
   renamed, line comments at every dispatch / wire / Win32-flag site.
   Tracked per binary in `scratch/annotate-progress/<binary>.json`.

## Worklists

Each binary's BFS worklist (sorted by reach + depth) is at
`scratch/annotate-worklist/<binary>.txt` — one address per line with
`reach` / `orphan` flag, depth, kind (`auto`, `thunk`, `named`), and
the current Ghidra symbol name.

## Resuming the pass

1. Open the target binary writable via
   `project_program_open_existing` (project root: `/home/gabriels/projetos/marvel-msn`,
   project: `MSN95`).
2. Read the binary's progress JSON; pop the next address from
   `next_priorities` or scan the worklist for the highest-priority
   FUN_ that is not yet in `deep_annotated`.
3. Open one transaction per function; rename, plate, line-comment,
   commit. The per-function workflow is in
   `.claude/skills/ghidra-annotate/SKILL.md` minus the skip rules
   (the user disabled them for this pass).
4. After each commit: append `{addr, name, session}` to the
   binary's `deep_annotated` list, decrement `remaining_FUN_`,
   update `lastUpdated`.
5. Periodically `program_save` the Ghidra DB.
